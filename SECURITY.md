# Security Policy

This repository contains research code and compact artifacts. It should not contain API keys, private data, model credentials, or raw benchmark dumps.

## Supported Versions

The `main` branch is the supported branch for security fixes.

## Reporting a Vulnerability

If you find a vulnerability, please do not post secrets or exploit details in a public issue. Use GitHub Security Advisories if they are enabled for the repository, or contact the maintainers through the contact information in the paper.

For ordinary reproducibility bugs, use the issue templates in `.github/ISSUE_TEMPLATE/`.
